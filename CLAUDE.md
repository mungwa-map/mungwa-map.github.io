# 문학 지도 — 이야기가 머무는 곳

한국 문학 작품의 배경이 된 장소들을 지도 위에서 탐험하고, 사용자가 직접 이야기를 남길 수 있는 웹앱.

## 기술 스택

- 순수 HTML / CSS / JavaScript (프레임워크 없음)
- SVG 기반 한국 지도 (현재: 17개 시·도, 목표: 시·군 단위)
- PWA 지원 (manifest.json, sw.js)
- localStorage로 사용자 글 저장
- Google Fonts (Noto Serif KR)

## 프로젝트 구조

- `index.html` — 메인 페이지 (지도 SVG, 스토리 패널, 글쓰기/상세보기 모달)
- `style.css` — 전체 스타일 (라이트/다크 테마 지원)
- `app.js` — 앱 로직 (지도 인터랙션, 패널, 모달, 테마, 상태 관리)
- `data.js` — 지역 정보 및 문학 작품 데이터
- `manifest.json` — PWA 매니페스트
- `sw.js` — 서비스 워커
- `icon.svg` — 앱 아이콘
- `PLAN.md` — 마스터 백로그 (전체 구현 계획)
- `PLAN-CURRENT.md` — 현재 진행 중인 항목
- `PLAN-DONE.md` — 완료된 항목 아카이브

## 주요 기능

- 지도에서 지역 클릭 → 해당 지역의 문학작품 목록 표시
- 작품 수 뱃지 표시
- 사용자 글 작성 (필명, 제목, 카테고리, 내용)
- 전체 / 문학작품 / 사용자 글 필터 탭
- 라이트/다크 테마 전환
- 모바일 대응 (터치, 드래그 핸들)

## 코딩 컨벤션

- 한국어 UI, 한국어 주석
- IIFE 패턴 (`app.js`)
- 빌드 도구 없음 — 파일을 직접 수정

## 콘텐츠 템플릿

### excerpt (인용문)

작품의 **실제 문장**만 사용. 줄거리 요약 금지.

- **시**: 대표 연(聯) 또는 첫 연. 2~4행
- **소설**: 유명한 첫 문장 또는 상징적 구절. 1~2문장
- 원문 줄바꿈은 `\n`으로 표기

### description (작품 설명)

**3문장 고정 구조** (탐방 정보 없으면 2문장):

1. **작품 소개** — 무엇을, 왜 쓴 작품인지 한 문장
2. **장소 연결** — 이 장소가 작품과 어떻게 연결되는지 한 문장
3. **탐방 정보** — 문학관·생가·기념비 등 방문 가능한 곳 (없으면 생략)

### location (장소)

`시·군 + 구체 장소` 형식:

```
춘천 실레마을
경주 불국사 일대
군산 내항 일대
서울 종로 미쓰코시 옥상
```

- 시·군은 필수, 뒤에 동·면이나 랜드마크
- 범위형(`공주 · 부여`) 금지 → 주 배경 한 곳으로 확정
- 괄호 보충(`순천 (무진)`) 금지 → description에서 설명

## 콘텐츠 작업 원칙

문학 작품 데이터를 만들거나, 수정하거나, 조사할 때는 **반드시 서브에이전트를 병렬로** 사용한다.

- **조사**: 작품·작가·장소 정보를 병렬 에이전트로 동시 검색 (WebSearch, WebFetch 활용)
- **생성**: excerpt 원문 검증 에이전트 + description 작성 에이전트를 분리 실행
- **검증**: 작성된 데이터의 사실 확인을 별도 에이전트로 교차 검증
- 단일 에이전트로 순차 처리하지 않는다. 항상 병렬 가능한 작업을 찾아 분리한다.

## 플랜 원칙

- "프로토타입 우선" 당장 정말 필요한 것부터 만든다. 초반부터 오버엔지니어링 금지
- 각 항목은 제목·헤딩 없이 `-`로 한 줄만 작성한다. 부가 정보 없음.
- 구체적 구현 계획은 `/start-phase`로 시작한 뒤 PLAN-CURRENT.md에서 진행.
- 완료 내용은 PLAN-DONE.md에서 관리한다.
- "무엇을 구현할지"가 아니라 생각해야 할 문제를 적는다.
- 구체적 구현 체크리스트는 PLAN-CURRENT.md에서 관리.

PLAN.md 예시:
```
- 44개 구역에서 원하는 지역을 빠르게 찾고 이동하는 경험
- 작품명이나 작가명만 알 때 해당 지역을 바로 찾아가는 방법
```

## 지역 세분화 계획

### 목표 단위

**시·군 단위** (문학적으로 배경이 뚜렷한 곳만 독립 구역으로)  
— 전국을 빠짐없이 쪼개는 게 아니라, 작품 배경이 있는 거점 위주로 선별  
— 목표 지역 수: **40~50개** 내외 (이 이상이면 SVG + 라벨이 지나치게 복잡해짐)  
— 배경 작품이 없는 시·군은 상위 도(道) 영역에 흡수

### 구역 설계 원칙

1. **문학 밀도** — 해당 시·군을 배경으로 한 작품이 1편 이상이면 독립 구역 고려
2. **지명도** — 독자가 지명을 들으면 작품이 연상될 정도의 상징성
3. **SVG 가독성** — 너무 작은 시·군은 인접 거점에 병합 (예: 세종 → 공주권)
4. **문학관·기념관 존재** — 실제 탐방 가능한 장소가 있으면 우선 독립

### 시·도별 예상 구역

| 시·도 | 독립 구역 | 근거 작품 |
|------|----------|---------|
| **서울** | 종로·중구 / 성북동 / 연희·신촌 / 마포 | 이상 「날개」, 이태준, 윤동주, 김수영 |
| **인천** | 인천 (현행 유지) | 개항기 근대 문학 |
| **경기** | 양평 / 파주 / 수원·화성 / 나머지 경기 | 황순원 「소나기」, 혜음원지 |
| **강원** | 춘천 / 평창·봉평 / 강릉 / 속초·양양 / 나머지 강원 | 김유정, 이효석, 허난설헌, 이순원 |
| **충북** | 옥천 / 청주 / 나머지 충북 | 정지용, 이문구 |
| **세종·대전** | 대전 / 세종·공주권 | 한용운(공주 마곡사) |
| **충남** | 부여 / 당진 / 아산·천안 / 나머지 충남 | 신동엽, 심훈, 윤봉길 |
| **경북** | 경주 / 안동 / 영양 / 문경·상주 / 나머지 경북 | 박목월·김동리, 이육사, 조지훈 |
| **대구** | 대구 (현행 유지) | 이상화 |
| **전북** | 군산 / 남원 / 고창 / 전주 / 나머지 전북 | 채만식, 최명희·춘향전, 서정주 |
| **광주** | 광주 (현행 유지) | 한강 「소년이 온다」 |
| **전남** | 벌교·순천 / 목포 / 해남 / 보성 / 나머지 전남 | 조정래, 김우진, 윤선도, 김승옥 |
| **경남** | 하동 / 통영 / 남해 / 나머지 경남 | 박경리 「토지」, 유치환·김춘수 |
| **울산** | 울산 (현행 유지) | 처용가 |
| **부산** | 부산 원도심·영도 / 해운대 | 최인훈 「광장」, 피란 문학 |
| **제주** | 제주시 / 서귀포 | 현기영, 이청준 |

→ 합계 약 **44개** 구역

### SVG 구현 방향

- 도(道) 영역은 현행 유지하되, 문학 거점 시·군은 **도 위에 겹쳐 그리는 레이어 방식** 유지 (현재 광역시 방식 그대로 확장)
- 시·군 폴리곤은 실제 행정구역 좌표 기반 대신 **문학적 인상을 살린 단순화 형태**로 손수 조정 (SVG viewBox 0 0 550 750 기준)
- 작은 시·군(영양, 고창 등)은 원형 또는 육각형 마커로 표현하는 방식도 검토

### 데이터 추가 우선순위

1순위 (작품·문학관 확실): 춘천, 평창(봉평), 강릉, 양평, 옥천, 부여, 경주, 안동, 군산, 남원, 고창, 하동, 통영, 벌교(순천), 목포
2순위 (작품 있으나 인지도 낮음): 영양, 당진, 해남, 아산, 파주, 남해, 속초
3순위 (확장 여지): 문경, 상주, 보성, 익산, 창원, 해운대

## 플랜 관리 흐름

플랜 파일 3종:
- `PLAN.md` — 마스터 백로그 (전체 구현 계획)
- `PLAN-CURRENT.md` — 현재 진행 중인 항목 (체크리스트 + 진행 로그)
- `PLAN-DONE.md` — 완료된 항목 아카이브

```
/start-phase 2-1
  │  PLAN.md에서 해당 항목 추출
  │  → PLAN-CURRENT.md에 체크리스트 생성
  │  → PLAN.md에 (진행 중) 표시
  │
  ▼  구현 작업 진행
  │
  │  [자동] Stop 훅 — 매 턴 종료 시
  │  index.html 변경 감지 → PLAN-CURRENT.md 업데이트 알림
  │
  ▼  모든 체크리스트 완료
  │
/complete-phase
     PLAN-CURRENT.md → PLAN-DONE.md에 아카이빙
     → PLAN.md에 [x] 체크 + 완료 날짜
     → PLAN-CURRENT.md 초기화
```

### 훅

| 이벤트 | 대상 | 동작 |
|--------|------|------|
| `PostToolUse` | `Edit\|Write` | CLAUDE.md 동기화 알림 |
| `Stop` | 전체 | PLAN-CURRENT.md 진행 상황 업데이트 알림 |
