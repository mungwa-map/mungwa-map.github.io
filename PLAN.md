# Development Plan

- 작품을 시대나 감정으로 묶었을 때 사용자에게 어떤 가치를 주는지
- 작품이 늘어났을 때 일관성을 유지하는 기준과 검증 방법
- 44개 구역에서 원하는 지역을 빠르게 찾고 이동하는 경험
- 작품명이나 작가명만 알 때 해당 지역을 바로 찾아가는 방법
- 시대별·감정별로 작품을 걸러볼 때 지도와 패널이 어떻게 연동되는지
- 작품이 많아졌을 때 스크롤, 정렬, 탐색이 자연스러운 구조
- 데이터 많아졌을 때 현재 구조로 가능할지?
- 사용자들이 실제로 많이 사용할 수 있게, 경쟁력 있게 하려면. 현실적 유입
- 앱에서 실제 방문으로 이어지는 동선을 어떻게 제공할지
- 내 위치 근처에 어떤 문학 장소가 있는지 바로 알 수 있는 방법
- 글을 쓰고 나서 다시 돌아오게 만드는 동기와 피드백 루프
- 목적 없이 들어온 사용자가 빠져들 수 있는 콘텐츠 소비 흐름
- 사용자가 실제로 접근할 수 있는 URL과 설치 가능한 PWA
