# Development Plan

> 큰 틀만 잡는 마스터 백로그. 각 항목은 2~5줄로 목표·방향을 설명.
> 구체적 구현은 `/start-phase`로 PLAN-CURRENT.md에서 진행.

---

## Phase 1: 지역 세분화 (1a 파일럿 완료 — 6곳, 나머지 확장 대기)

17개 시·도 → 약 44개 문학 거점으로 확장하는 핵심 작업.
다른 모든 Phase의 기반이 되므로 가장 먼저 진행.

### 1-1. 데이터 구조 개편
`REGIONS` 객체에 상위 도(道) 매핑(`parentRegion`)을 추가하고, 지역 ID를 시·군 단위로 세분화.
기존 `LITERARY_DATA`와 localStorage 사용자 글의 region 값도 새 ID에 맞게 마이그레이션.

### 1-2. SVG 지도 확장
CLAUDE.md의 시·도별 예상 구역표를 기준으로 시·군 path를 SVG에 추가.
1순위(춘천, 경주, 군산 등 15곳) → 2순위(7곳) → 3순위(6곳) 순서로 진행.
서울(4구역), 제주(2구역), 부산(2구역)은 기존 영역을 분할.

### 1-3. app.js 지도 로직 수정
새로 추가된 시·군 요소에 이벤트 바인딩, 뱃지 업데이트, 선택 하이라이트 등 기존 도(道) 로직을 확장 적용.
시·군 선택 시 상위 도도 연동 하이라이트할지 결정 필요.

## Phase 2: 콘텐츠 템플릿 표준화 (완료 — 2026-04-02)

지역·작품을 더 추가하기 전에 표현 형식을 먼저 확정.
템플릿이 없으면 데이터가 늘어날수록 품질이 들쭉날쭉해진다.

### 2-1. 작품 설명란 템플릿
각 작품 데이터(excerpt, description, location 등)의 톤·길이·구성을 획일화.
어떤 정보를 어떤 순서로, 얼마나 담을지 기준을 정하고 기존 데이터를 리라이트.

### 2-2. 패널·카드 UI 템플릿
우측 패널(전체/문학작품/사용자 글)의 카드 레이아웃과 상세 모달 구성을 구체화.
정보 배치, 표시 순서, 텍스트 길이 제한, 빈 상태 문구 등을 표준으로 잡는다.

---

## Phase 3: 데이터 확충

세분화된 지역에 콘텐츠가 있어야 의미가 있으므로 Phase 2 이후 진행.

### 3-1. 1순위 지역 작품 추가
춘천(김유정), 봉평(이효석), 강릉(허난설헌), 양평(황순원), 옥천(정지용),
부여(신동엽), 경주(박목월·김동리), 안동(이육사), 군산(채만식), 남원(춘향전·최명희),
고창(서정주), 하동(박경리), 통영(유치환·김춘수), 벌교·순천(조정래·김승옥), 목포(김우진).

### 3-2. 2·3순위 지역 작품 추가
영양(조지훈), 당진(심훈), 해남(윤선도), 파주(황석영), 속초(이순원), 남해(유배문학) 등.
3순위(문경, 보성, 해운대 등)는 여력 되면 추가.

### 3-3. 시대·감정 필드 추가
각 작품에 시대(`era`: 고전/개화기/일제강점기/해방~산업화/현대)와
감정 키워드(`emotions`: 그리움, 저항, 고독 등) 필드를 부여. Phase 4 필터의 기반 데이터.

### 3-4. 데이터 품질 점검
region 값 일치 여부, excerpt/description 길이 통일, 연도 누락·오류 확인.

## Phase 4: UX 개선

44개 구역이 되면 현재 UI로는 탐색이 어려우므로 줌, 검색, 필터 등이 필요.

### 4-1. 지도 줌/팬 + 툴팁
핀치 줌, 마우스 휠 줌, 드래그 팬. 줌 레벨에 따라 라벨 표시 전환.
호버 시 지역명 + 작품 수 툴팁.

### 4-2. 검색
헤더에 검색창. 작품명·작가명·지역명으로 검색.
결과 클릭 시 해당 지역 선택 + 작품 하이라이트.

### 4-3. 시대별 필터
Phase 3-3에서 추가한 `era` 기반으로 시대 필터 UI 추가.
기존 필터(전체/문학작품/사용자 글)와 조합 가능. 지도에도 해당 시대 지역만 강조.

### 4-4. 감정 키워드 필터
`emotions` 기반 컬러 칩 UI. 다중 선택(OR 조건).
작품 카드와 상세 모달에 감정 태그 표시.

### 4-5. 패널 개선
모바일 스와이프 제스처, 작품 정렬(연도순/가나다순), 작품 수 많을 때 대비.

### 4-6. 모바일 최적화
터치 영역 확대, 패널 애니메이션, iOS safe area, 가로 모드 대응.

## Phase 5: 콘텐츠 & 기능 확장

기본이 완성된 후 점진적으로 추가하는 부가 기능들.

### 5-1. 문학관/탐방 정보
작품 상세에 관련 장소(문학관, 생가) 섹션. 카카오맵/네이버맵 외부 링크 연동.

### 5-2. 사용자 기능 강화
사용자 글 수정, 좋아요/공감(localStorage), 내가 쓴 글 모아보기.

### 5-3. 탐험 기능
오늘의 작품 랜덤 추천, 작가별 모아보기, 문학 산책 코스(지역 간 경로).

### 5-4. 공유
Web Share API, URL hash 기반 딥링크, OG 메타태그.

---

## Phase 6: 기술 품질 & 배포

배포 전 품질 확보. Phase 4 이후, Phase 5와 병행 가능.

### 6-1. 접근성
SVG에 aria-label, 키보드 네비게이션, 스크린 리더 지원, WCAG AA 색대비.

### 6-2. 성능
SVG 좌표 최적화, 서비스 워커 캐싱 개선, Lighthouse 90+ 목표.

### 6-3. 배포
GitHub Pages 배포, PWA 래스터 아이콘(192px, 512px) 생성, 커스텀 도메인은 선택.

---

## 우선순위

1 → 2 → 3 → 4 → 6 → 5 (기반 → 템플릿 → 콘텐츠 → UX → 품질 → 확장)
