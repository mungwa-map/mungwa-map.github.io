# 현재 진행 중인 플랜

> 항목: Phase 1a / 지역 세분화 (파일럿)

## 목표

17개 시·도 지도에 파일럿 시·군 6개를 추가하여 "도 위에 시·군을 겹치는 패턴"이 잘 작동하는지 검증.
빠르게 동작하는 프로토타입을 만든 뒤, 나머지 지역은 Phase 1b에서 확장.

**파일럿 지역 (6곳):**
춘천 / 봉평(평창) / 경주 / 군산 / 하동 / 통영

선정 기준: 작품·문학관이 확실하고, 서로 다른 도(강원·경북·전북·경남)에 분포하여 레이어 겹침 패턴을 다양하게 테스트 가능.

## 체크리스트

### 1-1. 데이터 구조 개편
- [ ] REGIONS 객체에 `parentRegion` 필드 설계 추가 (시·군 → 상위 도 매핑)
- [ ] 파일럿 6개 지역 ID 및 메타데이터 추가 (chuncheon, bongpyeong, gyeongju, gunsan, hadong, tongyeong)
- [ ] 기존 LITERARY_DATA에서 해당 지역 작품의 region 값을 새 ID로 변경
- [ ] 기존 상위 도(gangwon, gyeongbuk 등) region으로 저장된 사용자 글은 그대로 상위 도에 표시 (별도 마이그레이션 불필요)

### 1-2. SVG 지도 확장
- [ ] 파일럿 6개 시·군 path를 SVG에 추가 (도 위 레이어 방식, 기존 광역시 패턴 그대로)
- [ ] 라벨 + 뱃지 SVG 요소 추가

### 1-3. app.js 지도 로직 수정
- [ ] 새 시·군 요소에 이벤트 바인딩 (기존 setupMap 로직 그대로 동작하는지 확인)
- [ ] 뱃지 업데이트 로직이 새 region ID도 커버하는지 확인
- [ ] 시·군 선택 시 상위 도 연동 하이라이트 처리

### 검증
- [ ] 각 파일럿 지역 클릭 → 패널에 작품 표시되는지 확인
- [ ] 상위 도 클릭 → 해당 도 소속 작품(시·군에 배정되지 않은 것) 표시 확인
- [ ] 뱃지 숫자 정확한지 확인
